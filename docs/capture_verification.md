# OBS Virtual Camera 実機動作確認手順

`src/nss_tracker/capture/ffmpeg_capture.py`(`FfmpegFrameReader`)は、`fixtures/videos/`の実動画ファイルを入力に差し替えたテストで「継続的にフレームを読み取る・最新フレームだけ保持する・停止する」という配管部分は検証済みだが、**OBS Virtual Camera実機(dshow経由)での疎通確認はまだ行っていない**。この手順書は、実際にOBS・Switchを起動した状態で動作確認する際の手順をまとめたもの。

## 事前準備

1. Switchとキャプチャーボード(I-O DATA GV-USB3HDS/E)を接続し、いつも通りプレイできる状態にする(HDMI OUT経由でプレイ用モニターにも映せる状態)
2. OBS Studioを起動し、キャプチャーボードの映像をソースとして追加する(いつもの配信・録画設定と同じでよい)
3. **OBSの「仮想カメラを開始」ボタンを押す**
   - これを押さないと、`ffmpeg`が掴もうとする`"OBS Virtual Camera"`というdshowデバイス自体が存在しない状態になる
4. OBSの映像設定(基本解像度・出力解像度)が**1920x1080**になっていることを確認する
   - コードは1920x1080固定で生バイト列を画像として組み立てているため、解像度が違うと映像が壊れる

## 実行方法

```bash
uv run python scripts/capture_stream_test.py 10
```

引数はフレームを取得し続ける秒数(省略時は5秒)。

## 確認ポイント

### 1. ターミナルの出力

- `取得フレーム数: ... (10.0秒, 実測 XX.Xfps)` が表示され、`フレームが取得できませんでした`というエラーが出ていないこと
- 実測fpsが極端に低い(1〜2fpsなど)場合は、何かがボトルネックになっている可能性がある(通常はOBSの出力フレームレート、30〜60fps程度に近い値が出るはず)

### 2. 保存された画像

リポジトリ直下に`capture_stream_test_last_frame.png`が保存される。これを開いて確認する。

- 実行時にOBSに映っていた画面(ゲーム画面やOBSのプレビュー)と内容が一致しているか
- **映像が斜めに流れていたり、色がおかしかったり、縦横比が崩れていたりしないか**(解像度不一致の典型的な症状。事前準備の4を再確認する)

## うまくいかない場合の切り分け

1. `uv run python scripts/list_dshow_devices.py` を実行し、`"OBS Virtual Camera"`という名前のデバイスが一覧に出てくるか確認する(OBSのバージョンによって名前が変わっていないか)
2. より単純な単発取得スクリプト `uv run python scripts/capture_test_frame.py` を先に試し、そちらが成功するか確認する(こちらは動作確認済みの実績があるため、これが失敗する場合はOBS側の設定(仮想カメラの起動忘れ等)に問題がある可能性が高い)
